03 Storage

Principle I · separate lifecycles

Two databases.
Different cadences.

Ephemeral operational state is pruned aggressively. Long-term memory is backed up, portable, vec-enabled. Soft references cross the boundary — deleting a session never erases what you learned.

04 Ingestion

Principle II · never wait for end-of-session

A book indexes
while you read.

POST /api/graph/ingest returns 202 Accepted in under 100 ms. Work drains in the background through two workers; unrelated reads stay responsive because SQLite runs in WAL mode and writes pool through r2d2.

Non-blocking

Work lives in the DB, not the channel. Worker restarts naturally recover in-flight pending episodes.

Backpressure

Global queue depth > 5,000 or per-source > 500 → 429 with retry-after. Books never starve chat.

06 One mechanism

Principle IV · stop writing cosine by hand

Every similarity query
goes through one index.

Phase 1b deleted six hand-rolled cosine_similarity functions and replaced them with sqlite-vec vec0 virtual tables — one per retrieval source. Embeddings live only in their vec partner. No dual storage. No drift.

07 Recall

Principle V · compete, don't append

Five sources.
One scored pool.

Facts, wiki articles, procedures, graph neighborhoods, and active goals all project to ScoredItem and merge via Reciprocal Rank Fusion — k = 60, same item across lists naturally rises.

08 Sleep-time

Principle VI · never hard-delete

The graph
stays lean,
forever.

Hourly tokio task or on-demand via POST /api/memory/consolidate. Every op records to kg_compactions. Archival entities (epistemic_class = 'archival') are exempt — historical facts never decay.

Soft-delete via compressed_into = '__pruned__' sentinel. Every recall query filters compressed_into IS NULL. Nothing is ever gone.
