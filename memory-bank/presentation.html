z-Bot

A goal-oriented AI agent that lives on your desktop.
Set a goal. Walk away. It figures out the rest.

⚡ Token-intensive by design — a single task can consume 100+ LLM calls across multiple agents
01

Most AI tools are built around control

You tell them what to do, step by step. Every action needs your approval. Every failure needs your intervention.

Constant Steering

You drive every step. The AI waits for instructions instead of solving problems.

No Memory

Every session starts from scratch. Past mistakes are repeated. Lessons are lost.

Single-Agent

One model tries to do everything. No specialization. No delegation. No parallelism.

03

What a session looks like

You say: "Build an auth system with JWT and role-based access"

🤖 root — analyzes intent, selects agents 2.1s
▶ planner-agent — "Create implementation plan" 8.4s · 3,200 tok
🔧 shell — ls src/ 0.2s
👁 read — src/main.rs 0.1s
💬 respond — "Plan: 4 steps..." 0.0s
▶ code-agent — "Implement auth middleware" 22.6s · 9,800 tok
✏️ edit — src/auth.rs 0.1s
🔧 shell — cargo test FAIL 1.8s
✏️ edit — src/auth.rs 0.1s
🔧 shell — cargo test PASS 1.6s
▶ tutor-agent — "Document the API" 6.2s · 2,100 tok
✏️ write — docs/auth-api.md 0.1s
💎 distill — 4 facts learned, 2 entities extracted 3.1s
04

It learns from every session

Five memory loops run automatically. No manual knowledge management.

1
System Recall — relevant facts injected at session start
2
Intent Enrichment — corrections and strategies inform intent analysis
3
Subagent Priming — relevant memory injected when spawning subagents
4
Mid-Session Refresh — new facts injected every N turns during long runs
5
Post-Session Distillation — LLM extracts facts, entities, relationships

06

Everything you need

🎯
Intent Analysis
LLM analyzes your message to determine complexity, select agents, and choose skills — before any work begins.
🤖
Multi-Agent Delegation
Root orchestrator delegates to planner, coder, researcher, tutor — each with their own tools and context.
🧠
Persistent Memory
Auto-distills facts from sessions. Recalls corrections, strategies, and domain knowledge. Knowledge graph for entity relationships.
📁
Code Wards
Persistent project directories. Agents create, name, and navigate them. Code survives across sessions.
🔌
Skills & MCP
Reusable instruction packages. Model Context Protocol for external tools — filesystem, databases, APIs.
📊
Observability
Timeline tree showing root → subagent → tool calls. See exactly what happened, how long it took, and what it cost.
