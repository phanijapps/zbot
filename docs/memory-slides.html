01  /  the metaphor

A transcript is not memory.

Every session is a page in the log — captured verbatim.

Memory is the reference book the agent writes from those pages. It's smaller, indexed, and the only thing read at the start of the next session.

every session = a page
memory = the reference book

02  /  the fragments

Memory is many small fragments,
not one big blob.

Each fragment has a typed prefix (correction, schema, handoff, goal) and a short opaque id.

Recall doesn't replay history — it pulls the handful of cards that matter for the current turn, ranked by similarity and category weight.

A correction is what you told it.
A schema is what it learned.
A handoff is what just ended.
A goal is what's in flight.

// phase 03 / 04

Conflict
resolution.

When two schemas disagree, only one wins.

Phase 03 · shipped

The notebook self-cleans

  • 01 Pair upFind schema pairs with cosine similarity ≥ 0.85 — likely about the same topic.
  • 02 Ask the judgeAn LLM decides whether they actually contradict, with a confidence score.
  • 03 Pick a winnerHigher confidence wins. Tiebreaker: more recently updated.
  • 04 Retire, don't deleteLoser gets superseded_by → winner. Hidden from recall, audit trail kept.
settings.json → execution.memory.conflictResolverIntervalHours
default: 24h · superseded fragments are excluded from recall.
Phase 04 · foundation shipped

Belief network — foundation shipped

Confidence now lives
on the things themselves.

Phase 4 lifts confidence off fragments and onto the knowledge-graph nodes — the people, projects, and concepts the fragments are about. Each entity and relationship now carries its own decaying score.

Temporal decay shipped: every sleep cycle, KG entity and relationship confidence is multiplied by 0.5^(days_since_last_seen / half_life), floored at min_confidence. An evidence column is in place on both tables, ready for the next pass.

Still to come: contradiction propagation from facts to KG nodes, and graph-traversal weighting by confidence.
